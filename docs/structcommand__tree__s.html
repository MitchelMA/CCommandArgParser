Detailed Description

This is the command_tree structure. This structure is the root of the tree, and holds all its direct descendands: commands.

Since you can't call a tree-root itself, there's no reason for this structure to hold a notation_s. For this reason I chose for the root to hold its own description, this description should be the description of the application itself.

For functionality and usage of this structure, look into the command_tree.h header-file.
